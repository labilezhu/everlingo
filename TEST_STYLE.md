# 测试风格

只为以下部分代码编写测试：
- 核心流程： `产品说明`，或 `产品设计` 相关文档有明确提及的流程
- 用户输入不同做成的边缘流程。如输入不合法，或输入格式不正确。如输入的配置项取值非法

其它规则：
- 不要为代码实现分支作过多的单元测试。除非是核心用户流程相关的代码。与用户输入无关的系统内部边缘错误处理不要写单元测试。
- 不要对 LLM Message 自然语言文本输出的任何部分作 assert 断定。

## Mock 的使用

### Mock LLM
- LLM 的调用，尽量 Mock

## Do NOT test
- 不要写强依赖于 LLM Message 自然语言文本输出 或 system prompt 自然语言文本输入的测试。如对 LLM Message 自然语言文本输出作字串上的匹配。因为这种测试很难应对变化，也很难维护。
